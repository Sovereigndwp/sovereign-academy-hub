◆Decision zero

Three directions to compare

The core tension: TSA's brand is dark graphite, but FSA lives on paper. Which world is canonical — the one tokens are defined in, and the other is derived from — shapes every downstream decision. Pick one before any component work starts.

DIRECTION A Recommended

Graphite Canon

Dark web is canonical; print is an export. Tokens defined on graphite, inverted for paper. Truest to today's TSA site.

Risk: FSA documents always feel like a dark website printed in a hurry. The anchor audience gets the derived experience.

DIRECTION B Recommended

Paper Canon

Warm paper is canonical — tokens defined light-first, B/W-photocopy-safe by construction. Graphite becomes TSA's and BSA's web skin, derived from the same scale.

Why it wins: FSA anchors the system. Anything born print-safe upgrades gracefully to screens; the reverse never holds.

DIRECTION C Recommended

Dual Native

Two co-equal themes; every semantic token ships a paired dark/light value. Nothing is derived — both are designed.

Risk: double maintenance for a one-person operation. Every new component needs two reviews forever.

The rest of this brief assumes the recommended direction. Everything else survives a switch; only §6 token definitions would flip.

01

Brand architecture

A branded house with endorsed programs. TSA is the masterbrand and the quiet authority behind everything; FSA and BSA are named programs that carry their own signal color and their own tone, endorsed by — never dressed as — the parent.

Parent

TSA — The Sovereign Academy

Holds the philosophy, the engine, and the credibility. Speaks in monochrome silver. Never competes with its programs for attention.

Endorsed program

FSA — Financially Sovereign Academy

Institution-facing. Its brand must survive a facility's photocopier, a district's procurement PDF, and a facilitator with no design tools.

Endorsed program

BSA — Bitcoin Sovereign Academy

Individual-facing. Precision and verification are the brand: exact numbers, exact steps, no crypto-generalization, no hype.

Endorsement grammar

Every program surface carries one — and only one — parent endorsement: the line A SOVEREIGN ACADEMY PROPERTY in mono caps, placed in the footer (web), the back cover (print), or the closing slide (decks). Program lockups never combine the TSA mark with the program name; the endorsement line is the whole relationship.

02

Visual relationship

Same skeleton, different metabolism. The three platforms share structure — type, spacing, component anatomy, the eyebrow-and-diamond section grammar — and differ in exactly two things: signal color and temperature.

TSA · Editorial monochrome

Cool, quiet, serif-led. Silver fade on a few key words. The most restrained of the three — the parent whispers.

FSA · Warm paper emerald

Light-first, generous whitespace, larger type, plainer words. The warm-educational half of the brand mood lives here.

BSA · Graphite ember

TSA's dark shell with the orange→yellow fade as its only heat. Mono-heavy: numerals, hashes, checklists, exactness.

Recognition test: any page from any platform, with logos removed, should be attributable to the family within two seconds — and to the right platform within five.

◆One skeleton, three skins

The same page, three ways

Identical anatomy in every column — eyebrow, heading with one fade phrase, body, callout, primary button, endorsement line. Only the signal color and the temperature change.

03

Shared design principles

P1

Color is meaning, not decoration

One signal color per platform, spent on a few words, one button border, one rule line. Neutral surfaces everywhere else. If a page has more than three colored moments, it's off-brand.

P2

Paper is a first-class citizen

Every artifact must pass the photocopier test: legible, complete, and correctly branded in one-color black on white. Anything that fails is a web-only ornament, not a system element.

P3

Plain language before ornament

Copy is part of the design system. Short sentences, concrete scenarios, no jargon without a glossary entry. FSA targets grade 6–8 reading level; TSA and BSA may run higher but never hide behind it.

P4

One skeleton, three voices

Structure is shared and non-negotiable; expression is per-platform and deliberate. A component may change its skin, never its anatomy.

P5

Show, don't hype

Numbers over adjectives, worked examples over promises, sources visible. Calm authority is the shared tone; urgency and FOMO are permanently off-brand — especially at BSA.

P6

Built for one maintainer

Every rule must be cheap to follow. Fewer tokens, fewer variants, no component that needs a designer to use correctly.

04

Always consistent

Type stack — Playfair Display / Inter / JetBrains Mono, same roles everywhere (§7)
Spacing & radii — 4px base grid; radii 2 / 4 / 8px only
Button anatomy — gradient-border outline, never hue-filled; 44px min height
Section grammar — mono-caps eyebrow with ◆ marker; serif heading with one italic fade phrase
Fade-on-words rule — brand fades land on words and thin rules, never on backgrounds or fills
Endorsement line — placement and typesetting per §1
Accessibility floors — contrast, sizes, targets per §15; no platform may trade them away
Diagram + icon style — one geometric line language per §13
05

Different by platform

Signal color — silver (TSA) / emerald (FSA) / orange→yellow (BSA); never mixed on one surface
Theme default — graphite web for TSA + BSA; warm paper light for FSA everywhere
Voice & reading level — editorial (TSA) / plain grade 6–8, scenario-first (FSA) / precise, imperative (BSA)
Density — FSA runs ~15% larger type and looser leading; BSA runs the densest, with mono data blocks
Signature components — scenario worksheets (FSA) / custody checklists & verification blocks (BSA) / property cards (TSA)
Imagery — TSA: abstract/architectural; FSA: documentary photos of real people and paper; BSA: hardware, hands, keys — never coins-and-rockets stock art
Motion budget — TSA modest, BSA functional-only, FSA near-zero (§14)

07

Typography system

One stack, three roles, all platforms. This delivers the modern-editorial half of the mood (serif display, mono labels) while Inter keeps body text warm and effortless to read.

Playfair Display
Display · headings · pull quotes

Weights 500/700/900 + italic. Italic reserved for the one fade-accented phrase per heading. Never below 19px.

Inter
Body · UI · forms · captions

400–700. Web body 17px/1.6 (FSA: 18px/1.65). The workhorse — everything not a heading or a label.

JetBrains Mono
Eyebrows · data · labels · addresses

Mono-caps with 0.18–0.25em tracking for section tags. At BSA it also sets amounts, fees, seed-phrase blanks, and anything verifiable.

Print & institutional fallbacks

Documents produced outside our tooling (facilitator-edited Word docs, facility print shops) fall back to Georgia / system sans / Courier New in the same roles. Templates must be designed so the fallback version is still on-system: role and hierarchy carry the brand, not the specific font file. Print body minimum 12pt; FSA participant materials 13pt.

08

Component library

Built in three tiers. Every component ships in graphite, paper, and one-color-print skins from day one — a component without a print skin doesn't merge.

TIER 1 · CORE

Buttons (primary fade-border, secondary, text) · cards · eyebrow/section-tag · callout (note / caution / stop) · step list · definition/glossary term · data table · progress marker · footer with endorsement line

TIER 2 · LEARNING

Module header · lesson objective block · knowledge check (multiple choice, true/false) · scenario block (setup → decision → outcome) · reflection prompt with writing space · vocabulary sidebar · "try it" interactive frame

TIER 3 · PLATFORM SIGNATURE

FSA: budget worksheet grid, facilitator cue box, session agenda · BSA: custody checklist, verification block ("check this yourself"), key-ceremony steps, inheritance worksheet · TSA: property card, principle grid

09–12

Template families

Four families, one grid. Documents and workbooks are FSA-anchored (US Letter, one-color-safe); slides and web pages inherit the same section grammar at their native sizes.

09 · DOCUMENTS

US Letter, 0.75in margins, single column with a 1.75in sidebar zone for cues and definitions. Templates: one-pager · facilitator guide · participant handout · partner/procurement brief · letterhead.

Rules: 12pt minimum, no color-dependent meaning, header carries program name + module code in mono, footer carries page number + endorsement line. Must photocopy clean at 100% black.

10 · SLIDES

1920×1080, six masters: title · section divider · content · scenario · diagram · quote/close. Body text never under 24px; one idea per slide.

TSA/BSA decks run graphite; FSA decks run paper-light by default (classrooms, projectors, printouts of decks). Section dividers are the only slides that may use a fade rule full-width.

11 · WORKBOOKS

FSA's flagship family. Pages: cover · module opener · lesson spread · scenario worksheet · fill-in tables · reflection page · glossary · completion certificate.

Writing-space rules: 0.4in line spacing for handwriting, generous answer boxes, checkbox size ≥ 0.18in. No content that requires a device, a link, or a QR code to complete. Spanish-first variants planned from the start, not retrofitted.

12 · WEB PAGES

Five templates: landing · learning-path overview · lesson/module · interactive-demo shell · institutional program page (FSA's partner-facing pitch).

All share the 1100px wrap, sticky nav, band-section rhythm, and footer grammar of the current TSA site. Lesson pages get a persistent progress marker and a print-this-lesson path that emits the §9 document layout.

13

Visual & diagram style

Line language

Geometric line diagrams: 1.5px strokes, square or 4px-rounded nodes, mono labels, straight or 90° connectors. Ink-colored lines; the platform signal color marks exactly one thing — the decision point, the flow of money, the key. No 3D, no isometric, no clip art, no mascots.

Icons

One outline icon set (1.5px stroke, 24px grid) shared by all platforms, always paired with a text label in learning contexts. No emoji anywhere in the system.

Photography

Documentary, not stock: real hands, real paper, real hardware, natural light, desaturated ~10%. Photography is optional everywhere and forbidden as a meaning-carrier — every photo must be removable without losing information (photocopier test again).

14

Motion rules

Motion explains; it never performs. Global budget: 150–250ms, ease-out, opacity + ≤4px translate. Hover lift ≤3px. No parallax, no scroll-jacking, no autoplaying loops, no animated gradients.

TSA — full (modest) budget: section reveals on first load, card hovers, fade-rule draw-in on hero only.
FSA — near-zero: state changes only (focus, expand/collapse, form validation). Institutional screens must read as still documents.
BSA — functional only: simulation feedback, step-completion ticks, confirmation counts. Motion may demonstrate a mechanism (blocks confirming) but never celebrates.
All — prefers-reduced-motion honored completely: everything lands instantly, nothing is lost.
15

Accessibility & print rules

Floors — every platform

WCAG 2.2 AA · text contrast ≥ 4.5:1 (fade text measured at its weakest stop) · web body ≥ 17px · touch targets ≥ 44px · full keyboard paths · visible focus (2px silver ring) · color never the sole carrier of meaning · alt text on every informative image

Print gates — FSA-anchored

One-color photocopy test is a release gate · 12pt minimum (13pt participant materials) · no information in color, links, or QR codes alone — every URL also written out · works stapled, hole-punched, and copied at 90% · facilitator versions annotate in the sidebar zone, never inline

Language

FSA: grade 6–8 reading level, checked per document · scenario-first explanations · glossary entry for every term of art · Spanish parity for FSA program materials · BSA may be technical but defines every term on first use

16

Migration plan

Tokens before components, components before pages, paper before pixels. Each phase ships something usable on its own; nothing waits for a big-bang relaunch.

PH 1
Tokens & type

Ratify Direction B. Publish the token sheet (both scales, three signals, semantic mapping), the type scale, and the spacing/radius grid as a single reference page. No product changes yet.

PH 2
FSA print kit first

Documents + workbook templates (§9, §11) — the anchor audience and the strictest constraints. Pilot with one institutional partner before locking the family.

PH 3
Tier-1 web components + slides

Core component set in all three skins; the six slide masters. BSA adopts the shell first (closest to current TSA styling), FSA's light web theme derives from the paper canon.

PH 4
Page templates & TSA refresh

The five web templates roll out per-page, highest-traffic first. TSA's own site is last — the parent adopts the system only after both programs prove it.

17

What not to redesign yet

Live curriculum in use — the proven programa-colombia materials and anything currently in facilitators' hands. Reprint on new templates only at natural content-revision moments.
BSA interactive demo internals — the 50+ simulations keep their working UIs; only their outer shell (nav, type, frame) adopts the system in Phase 3.
Logos and marks — the current marks stay until the system settles; a mark refresh, if any, is its own project after Phase 4.
Information architecture & URLs — this system restyles pages; it does not reorganize them. IA changes ride separately.
Advisor Certification brand — deliberately out of scope; it gets its own brand on this engine in a later phase.
